Abstract

Traditional video surveillance requires constant human monitoring, which is inefficient and prone to error. While streaming 24/7 video to the cloud for AI analysis provides high accuracy, it incurs massive bandwidth and API costs. Identifying real-time threats using a hybrid machine learning pipeline is a highly promising approach.

We propose a model that incorporates Edge Computing (OpenCV) for zero-cost motion filtering, paired with Cloud AI (Multi Model LLM) for deep reasoning. By calculating pixel differences to detect structural changes locally, the system isolates critical frames.

Based on our implementation, this multimodal approach autonomously identifies complex societal threats (fights, fires, accidents), extracts text (OCR) like license plates, and logs structured JSON data to a local Streamlit Dashboard with automated SMTP alerts.

Introduction

  • The Core Problem: Standard object detection models (like basic YOLO) track items (e.g., "person", "car") but lack contextual reasoning. They cannot easily distinguish between a normal human interaction and a violent anomaly (like a physical assault).
  • The Cost Barrier: Processing continuous video streams (30 fps) on cloud APIs is financially unscalable and consumes massive bandwidth for 24/7 security systems.
  • Data Overload: Security personnel suffer from fatigue when monitoring blank screens. Critical incidents often go unnoticed until after the fact.
  • The Innovation: The introduction of Multimodal Large Language Models (LLMs) allows us to query an image with natural language. By coupling this with an edge-node motion filter, we create a highly accurate, cost-effective prediction model.

Existing System

In existing systems, the implementation of Deep Learning for anomaly detection is highly complex and resource-intensive. Current models rely on continuous video streaming to cloud servers or require extremely expensive local GPUs to run heavy CNN architectures 24/7.

Furthermore, traditional systems require separate models for different tasks (one model for object detection, one for action recognition, and one for OCR/license plates), increasing latency.

Disadvantages of Existing Systems:

  • High computational complexity and hardware costs.
  • Massive network bandwidth consumption.
  • High false-positive rates due to lack of contextual understanding.

Proposed System

We propose a unified, hybrid model. At the Edge Node, a lightweight Python script uses OpenCV (absdiff, GaussianBlur, findContours) to continuously monitor pixel changes. This acts as a zero-cost gatekeeper.

When anomalous motion exceeds a set threshold, the system extracts the critical frames and securely dispatches them to the Multi Model LLM. It utilizes its multimodal vision-language capabilities to evaluate the scene.

The AI returns a strict JSON payload containing the threat type (Violence, Fire, Intrusion) and any extracted text (OCR). The system immediately logs this to a local SQLite database and dispatches an SMTP email alert.

Advantages of Proposed System

  • High Accuracy: Leverages state-of-the-art LLM reasoning rather than rigid bounding boxes.
  • Drastically Reduces Cost: API calls are only made when genuine motion occurs, saving 99% of bandwidth.
  • All-in-One Multimodality: Performs anomaly detection, contextual description, and OCR simultaneously.
  • Real-time Dashboarding: Provides operators with an organized, queryable SQLite log via Streamlit.

Implementation

  • Step 1: Initialize the local SQLite Database (security_log.db) and start the Streamlit UI.
  • Step 2: Connect to the camera feed (via IP Webcam/Droidcam) using cv2.VideoCapture.
  • Step 3: Continuously calculate the absolute difference between sequential frames. Apply Gaussian blur and thresholds to isolate moving contours.
  • Step 4: If contour area > 8000, trigger the AI sequence. Apply a 15-second cooldown to prevent spam.
  • Step 5: Convert the frame to a PIL Image and dispatch to the Multimodel LLM with a strict JSON formatting prompt.
  • Step 6: If threat_detected is True, insert the record into the database and spawn a background thread to send the SMTP alert with the image attached.

Module 1: Edge Processor

  • Video Ingestion: Connects to IP cameras via cv2.VideoCapture.
  • Grayscale Conversion: Simplifies the image matrices to reduce compute load.
  • Gaussian Blurring: Smooths out high-frequency noise (like wind or camera jitter) that causes false positives.
  • Thresholding & Dilation: Converts the difference image to stark black and white, making human/vehicle shapes distinct.
  • Contour Area Filtering: Ignores small movements (leaves, bugs) and only flags significant motion (area > 8000).
  • Cooldown Logic: Uses time.time() to enforce a 15-second delay between triggers to prevent API throttling.

Module 2: Cloud AI & Dashboard

  • Payload Generation: Converts the triggered OpenCV frame into a PIL Image format compatible with Google GenAI.
  • Prompt Engineering: Forces the LLM to output a strict JSON string mapping to specific database columns (threat_detected, threat_type, description, extracted_text).
  • SQLite Integration: Opens a local database connection and inserts the parsed JSON data alongside a real-time timestamp.
  • Multithreaded SMTP: Triggers the email.message library in a separate threading.Thread so the camera feed doesn't freeze while sending the attachment.
  • Streamlit UI: A web interface that reads the SQLite incidents table and uses st.dataframe to display a live, structured log for security personnel.

Conclusion

This project successfully demonstrates that high-end enterprise security monitoring does not require massive cloud computing budgets or continuous data streaming.

By blending the lightweight, zero-cost processing speed of local OpenCV on the Edge with the deep contextual understanding of Multimodal LLMs, the system can autonomously identify complex societal threats with near-zero false positives.

The integration of native OCR, structured JSON output, local SQLite database logging, and automated SMTP threading creates a highly secure, privacy-first, and fully functional real-time anomaly detection pipeline.
